# Glossary

This glossary standardizes terminology used across the documentation.

---

## A

**AppStorage**

A SwiftUI property wrapper that syncs a value with `UserDefaults`.

**ATS (App Transport Security)**

Apple’s default network security policy requiring HTTPS unless exceptions are configured.

---

## C

**Caches Directory**

A directory intended for purgeable, non-critical files (iOS may delete it).

**Core Data**

Apple’s persistence framework for structured data storage, querying, and migrations.

---

## D

**Dependency Injection (DI)**

A technique where dependencies are provided from the outside rather than created inside a type.

**DIContainer**

The app’s centralized dependency wiring object.

---

## M

**MVVM**

Model–View–ViewModel.

- Model: data structures
- View: SwiftUI UI layer
- ViewModel: state + orchestration

---

## N

**NetworkService**

A shared service responsible for making HTTP requests and decoding responses.

---

## O

**ObservableObject**

A Combine-compatible type that publishes changes for SwiftUI to observe.

---

## P

**Published**

A property wrapper that publishes changes to subscribers.

---

## R

**Role-based routing**

UI routing decisions based on the user’s role (student/teacher/admin).

---

## S

**Service**

A type that encapsulates a feature domain’s API surface.

**Socket.IO**

A real-time communication protocol/library used for events like session termination and messaging.

---

## V

**ViewModel**

A testable state holder that coordinates between Views and Services.

---

## W

**WindowGroup**

SwiftUI scene container that holds the main app window.
